Futures Scalper: Build From Scratch
Nine prompts to build a regime-aware intraday futures bot with Claude Code. Sim broker by

Guide: Read Me First
This guide builds a futures scalping bot from scratch in 9 prompts you paste into Claude Code one at a time.
The bot detects the intraday volatility regime with a Hidden Markov Model, runs a scalping playbook that
fits that regime, sizes everything in contracts, and enforces prop-firm rules so a funded account stays inside
its daily loss limit and trailing drawdown. It ships with a simulation broker so it runs with no account, and it
plugs into IBKR, TradeStation, or a prop gateway like Topstep when you want to go live.
Be clear about the scope. This is regime-aware intraday trading on short timeframes. It is not tick-by-tick
HFT, and it does not pretend to be. The HMM reads how violent the tape is right now and decides which
setups are allowed and how big. The setup decides long or short.
You do not need to know Python. Claude Code writes all of it. You just need to follow along so you can tune
it later.
Setup Requirements
• VS Code, from code.visualstudio.com
• The Claude Code extension from the VS Code marketplace
• Python 3.10 or newer, from python.org
• A futures account when you are ready for live: IBKR, TradeStation, or a prop firm like Topstep. You
do not need one to start. The simulator covers everything until then.

• Run the verification step after each prompt. If something breaks, paste the error back to Claude Code

STEP 3 · Intraday Features and the Regime HMM
What this does: The brain. It clusters intraday bars into volatility regimes using only past data, so the
regime printed at a bar never changes when later bars arrive.
The Prompt
Copy and paste this into Claude Code:
Implement core/features.py and core/hmm_engine.py.
core/features.py:
 Pure indicator functions on OHLCV: log returns (1, 5, 15), realized vol,
 vol ratio (short/long), normalized volume z-score, volume trend slope, ADX(14),
 SMA slope, RSI(14), distance from 200 SMA, ROC (10, 20), normalized ATR.
 Session-relative additions for futures:
 session_vwap (anchored, resets each day), distance_from_vwap,
 time_of_day_sin_cos (cyclical encoding so 23:59 and 00:01 are close).
 rolling_zscore(series, window) for standardisation.
 FeatureEngineer:
 HMM_FEATURES = the 17 standardised columns including dist_vwap, tod_sin,
 tod_cos.
 compute_hmm_features(bars): build raw features, z-score everything except the
 already-cyclical time features, return the 17-column frame, dropna.
 compute_strategy_features(bars): un-standardised close, ema_9, ema_21, atr,
 rsi, adx, vwap, roc_5 that the strategies read directly.
core/hmm_engine.py:
 Gaussian HMM. Sort and label regimes by VOLATILITY, not return, because for
 intraday futures the question is how violent the tape is. Labels ascending by
 vol: 3 -> [QUIET, NORMAL, VOLATILE], 4 -> add ACTIVE, 5 -> add EXTREME.
 Store expected_return per regime too so strategies can read a directional lean.
 fit(features): BIC model selection across n_candidates with n_init restarts,
 pick lowest BIC, build the raw-state to sorted-by-vol label map.
 REGIME DETECTION, NO LOOK-AHEAD. Do NOT use model.predict (Viterbi revises past
 states with future data). Implement the forward algorithm in log space:
 framelogprob = model._compute_log_likelihood(X)
 log_alpha[0] = log(startprob) + framelogprob[0]
 log_alpha[t, j] = logsumexp(log_alpha[t-1] + log(transmat[:, j])) +
 framelogprob[t, j]
 normalise each row to a posterior; reorder columns to the vol-sorted order.
 filtered_posteriors(features) returns per-bar posteriors in one pass.
 predict_regime_filtered(features) returns the last bar's RegimeState with the
 stability filter applied.
 MANDATORY look-ahead test in tests/test_look_ahead.py: the filtered posterior at
 bar t computed from features[0:t] must equal the value at t computed from
 features[0:t+k], to 1e-9. If it fails, the forward pass is peeking, fix it.
 Stability filter: a new regime is confirmed only after it persists
 stability_bars bars; until then hold the previous confirmed regime and mark the
 state unconfirmed. Track a flicker count over flicker_window; is_flickering()
 when it exceeds flicker_threshold.
 RegimeInfo (regime_id, regime_name, expected_return, expected_volatility,
 vol_rank 0..1), RegimeState (label, state_id, probability, distribution,
 timestamp, is_confirmed, consecutive_bars), TrainingMetrics.
 save/load with pickle.

STEP 4 · Scalping Signal Engine
What this does: The playbook. Each regime gets a strategy that fits it, and trades go both long and short.
The Prompt
Copy and paste this into Claude Code:
Implement core/scalp_strategies.py.
The regime layer does not call direction tick by tick. It reads the volatility
environment and decides which playbook is allowed and how big. The setup decides
long or short. Map by volatility rank, same idea as the stock template:
 rank = regime_id / (n_regimes - 1)
 rank <= 0.33 -> QuietRangeScalp
 rank >= 0.67 -> VolatileBreakoutScalp
 label EXTREME -> StandAside
 otherwise -> NormalTrendScalp
Direction enum (LONG, SHORT, FLAT). Signal dataclass: symbol, direction,
confidence, entry_price, stop_price, target_price, stop_ticks, target_ticks,
regime fields, timestamp, reasoning, strategy_name, metadata.
BaseScalpStrategy(config, regime_info) with generate_signal(symbol, feats,
regime_state, instrument) -> Optional[Signal], and a _build helper that places
the stop and target on the correct side and rounds to the tick.
Strategies (all read compute_strategy_features, sizes in ticks via ATR):
 QuietRangeScalp (low vol): mean reversion. Measure stretch = (close - vwap)/atr.
 Short when stretch > band and RSI > 60; long when stretch < -band and
 RSI < 40. Tight stop and target.
 NormalTrendScalp (mid vol): momentum. Require ADX >= adx_min. Long when
 ema_9 > ema_21 and roc_5 > 0; short on the mirror. Wider target than stop.
 VolatileBreakoutScalp (high vol): breakout continuation through the
 breakout_lookback high or low, widest stop.
 StandAside (EXTREME): return None, take no new risk.
stop and target come from stop_atr / target_atr times current ATR, converted to
ticks, with a sane floor.
ScalpOrchestrator(config, regime_infos): bind one strategy instance per regime by
vol rank; generate_signal dispatches to the strategy for the current regime;
strategy_name_for(regime_id) for logging.
How to Verify It Worked
Ask Claude Code to write tests/test_scalp_strategies.py and run:
python -m pytest tests/test_scalp_strategies.py -v
Check that a long signal has its stop below entry and a short has its stop above, and that the EXTREME
regime returns nothing.
Customization
stop_atr and target_atr set your risk-reward per trade. The defaults aim for roughly one to two. Tighten
stop_atr for smaller risk per trade, widen target_atr if you want to let winners run.

STEP 5 · Contract Risk and Prop-Firm Rules
What this does: The gatekeeper. Sizes positions in contracts and enforces the funded-account rules that
decide whether you keep the account.
The Prompt
Copy and paste this into Claude Code:
Implement core/risk_manager.py.
The risk manager has absolute veto and runs independently of the HMM. Even if the
regime is wrong, the breakers cap the damage off actual P&L.
Sizing in contracts:
 stop_ticks = instrument.stop_distance_ticks(entry, stop)
 risk_dollars = equity * risk_per_trade * breaker_size_multiplier
 per_contract_risk = stop_ticks * tick_value
 contracts = floor(risk_dollars / per_contract_risk)
 cap by margin: floor(equity * max_margin_utilization / day_margin)
 cap by max_contracts. Reject if stop is tighter than min_stop_ticks.
Prop-firm rules (first-class circuit breakers), all from config:
 daily_loss_limit: when session_start_equity - equity >= limit, HALT the session.
 daily_loss_reduce_at: at this fraction of the limit, halve size.
 trailing_max_drawdown: trail the high-water mark. If equity drops a breach below
 (high_water_mark - trailing), LOCK the account, write trailing into a lock file
 that must be deleted by hand to resume. If trailing_locks_at_start, the floor
 never drops below starting_equity - trailing.
 max_daily_trades cap.
Dataclasses: AccountState (equity, starting_equity, session_start_equity,
high_water_mark, open_contracts, trades_today, realized_pnl_today),
RiskDecision (approved, contracts, sized_signal, rejection_reason, modifications),
BreakerState (level, reason, triggered_at, size_multiplier),
HaltLevel enum (NONE, REDUCED, HALTED_SESSION, HALTED_LOCKED).
Methods:
 size_contracts(account, signal, instrument) -> (contracts, notes)
 validate_signal(account, signal, instrument) -> RiskDecision. Reject if locked
 or halted, if over the daily trade cap, if the stop is missing or on the wrong
 side, or if it sizes to zero. FLAT signals are always approved.
 update_after_fill_or_mark(account): maintain the high-water mark, then check
 trailing drawdown (lock), then daily loss (halt), then reduce threshold.
 start_session(account): reset session equity, trade count, clear a session halt.
 is_halted(), check_lock_file().
Set the prop-firm defaults as placeholders and say plainly in the config that the
user must put their account's real numbers in before trading funded.

What this does: Ties it together into a session-aware loop that runs the pipeline and places orders, with the

The Prompt
Copy and paste this into Claude Code:
Implement main.py with four modes: live, backtest, validate, train-only.
Shared: load_config, pick the primary symbol from universe.symbols, resolve the
instrument with any config overrides, set up logging.
train-only: load bars from data.source, compute features, fit the HMM, save it.
live(demo, fast):
 Build the broker from config (sim default), connect. Load the saved HMM or train
 and save one if missing. Build the orchestrator and risk manager. If a lock file

 Bar source: with the sim broker (or --demo) replay the loaded data source through
 the loop; with a live broker, poll get_bars each interval and gate on the session
 calendar (skip when closed, respect the maintenance break).
 Each step:
 1. With sim, call on_bar to settle bracket exits and log them.
 2. New session at a date change: risk.start_session.
 3. Sync equity from the broker, run risk.update_after_fill_or_mark.
 4. Filtered regime from features up to now. If HALTED_LOCKED, flatten, alert,
 and stop entering.
 5. If not halted, regime confirmed, probability >= min_confidence, not
 flickering, and flat in this symbol: get a signal, validate it, and if
 approved place a bracket order, increment the trade count, fire a Telegram
 alert, and log the entry.
 6. Render the dashboard.
 On Ctrl-C: stop the dashboard and disconnect. Positions keep their stops; do not
 market-close on shutdown.
backtest and validate: delegate to backtest/ (next prompts). Wire the CLI:

Options Fundamentals
Understanding the building blocks of every options strategy
What Is an Option?
A contract giving the right — but not the obligation — to buy or sell
An option is a contract between a buyer and a seller. The buyer pays a premium for the right to buy (call) or sell
(put) an underlying asset at a specific price (strike) before a specific date (expiration). The seller collects the
premium and takes on the obligation.
CALL OPTION
Right to BUY the underlying at the strike price
Buyer profits when price rises above strike + premium
paid. Seller profits when price stays below strike. Most
common bullish bet.
Example: Buy SPY $580 Call for $3.50
Breakeven: $583.50
Max loss: $350 (the premium)
PUT OPTION
Right to SELL the underlying at the strike price
Buyer profits when price falls below strike − premium
paid. Seller profits when price stays above strike. Most
common bearish bet or hedge.
Example: Buy SPY $570 Put for $4.00
Breakeven: $566.00
Max loss: $400 (the premium)
Buyer vs. Seller: Rights & Obligations
The fundamental asymmetry that defines every options trade
OPTION BUYER (Long)
→ Pays premium upfront
→ Has the RIGHT to exercise
→ Max loss = premium paid (defined)
→ Max profit = unlimited (calls) or strike − premium (puts)
→ Wants big moves in their direction
→ Time decay works AGAINST them
→ Benefits from rising IV
OPTION SELLER (Short)
→ Collects premium upfront
→ Has the OBLIGATION if assigned
→ Max profit = premium collected (defined)
→ Max loss = unlimited (naked) or defined (spreads)
→ Wants price to stay in range
→ Time decay works FOR them
→ Benefits from falling IV
~70-80% of options expire worthless or are closed before expiration. This is why selling premium is the foundation of income strategies — but
the risk is asymmetric. One bad trade can wipe out months of collected premium.
Moneyness: ITM, ATM, OTM
Where the strike sits relative to the current price
IN THE MONEY (ITM)
Call: strike < current price
Put: strike > current price
Has intrinsic value. Higher premium, higher delta, more expensive. Behaves
more like the underlying stock. Used for deep ITM covered calls or LEAPS
replacement strategies.
AT THE MONEY (ATM)
Call: strike ≈ current price
Put: strike ≈ current price
Maximum extrinsic (time) value. Delta near 0.50. Highest theta decay. Most
sensitive to IV changes. Common for straddles and strangles.
OUT OF THE MONEY (OTM)
Call: strike > current price
Put: strike < current price
No intrinsic value — all extrinsic. Lower premium, lower delta, cheaper. Most
commonly sold for income strategies (iron condors, credit spreads). Higher
probability of expiring worthless.
Option Pricing: Intrinsic vs. Extrinsic Value
What you're actually paying for when you buy an option
Option Premium = Intrinsic Value + Extrinsic Value (Time
Value) The price of an option is the sum of its intrinsic value (what it's worth if exercised now) and extrinsic value (what you're paying
for time and uncertainty).
INTRINSIC VALUE
The real, tangible value — what the option is worth if
exercised right now.
Call: max(0, stock price − strike)
Put: max(0, strike − stock price)
Only ITM options have intrinsic value. ATM and OTM
options have zero intrinsic value.
EXTRINSIC VALUE (TIME VALUE)
The speculative premium — what you're paying for the
possibility of future movement.
Driven by: time to expiration, implied volatility, interest
rates, and dividends.
Decays over time (theta). Highest for ATM options. Goes to
zero at expiration.

Expiration Cycles & Time Decay
When options expire and how time affects their value
Weekly (0-7 DTE)
Fastest theta decay. Cheapest premium. Popular for 0DTE plays and quick income. Gamma risk is extreme near expiry — small price
moves cause huge delta swings. Not for beginners.
Monthly (15-45 DTE)
The sweet spot for most income strategies. Theta decay accelerates around 45 DTE. Iron condors and credit spreads typically target
30-45 DTE entry. Enough time for adjustments if trade goes against you.
Quarterly / LEAPS (90-730 DTE)
Slowest decay, highest premium. LEAPS (>1 year) used as stock replacement strategies — buy deep ITM calls instead of shares for
leverage with defined risk. Quarterly for longer-term directional bets.
Theta decay is non-linear — it accelerates as expiration approaches. This is why sellers love 30-45 DTE entries (capturing the curve) and buyers
hate holding past 21 DTE.
Exercise, Assignment & Settlement
What actually happens when options expire or get exercised
American Style Can be exercised ANY time before expiration. All US equity options are American style.
Early assignment risk exists for short options, especially near ex-dividend dates or deep
ITM.
Most stock options
European Style
Can ONLY be exercised at expiration. Index options (SPX, NDX, RUT) are European style.
No early assignment risk, which simplifies position management.
Index options (SPX, NDX)
Cash Settlement
No shares change hands. The difference between strike and settlement price is paid in
cash. SPX and other index options settle in cash — no assignment of shares.
SPX, NDX, VIX options
Physical Settlement
Actual shares are bought/sold at the strike price. If your short put is assigned, you buy
100 shares at the strike. This is the basis of the wheel strategy.
Stock options (SPY, AAPL)
Contract Specifications
The numbers behind every options trade
Contract Multiplier 1 contract = 100 shares. A $3.50 option costs $350. A $0.05 move = $5. This multiplier is why
options provide leverage — and why losses can add up fast.
Tick Size $0.01 for options priced above $3, $0.05 for options below $3. The tick size affects your fills
and the minimum spread width.
Strike Increments $1 increments for most stocks, $0.50 for high-volume names (SPY, QQQ, AAPL). $5 for lowerpriced stocks. Narrower strikes = more precision.
Margin Requirements Naked options require significant margin. Defined-risk spreads use the width of the spread as
collateral. Cash-secured puts require the full strike × 100 in cash.
Pattern Day Trader (PDT) Accounts under $25k are limited to 3 day trades per 5 business days. Options count. Spreads
opened and closed same day = 1 day trade per leg.

The Greeks
The sensitivity measures that drive every aspect of options pricing and risk
The Greeks: Overview
Five numbers that tell you everything about your option's behavior
The Greeks measure how an option's price changes in response to different market variables. They're not theoretical
abstractions — they're the math behind every P&L move in your portfolio.
Δ Delta Price sensitivity How much the option price moves per $1 move in the underlying. Also
approximates probability of expiring ITM.
Γ Gamma Delta sensitivity How fast delta changes per $1 move. Highest at ATM near expiry. The
"acceleration" of your position.
Θ Theta Time sensitivity How much value the option loses per day from time decay. Always negative for long
options, positive for short.
V Vega Volatility sensitivity How much the option price changes per 1% move in implied volatility. Highest for
ATM, long-dated options.
ρ Rho Interest rate sensitivity How much the option price changes per 1% move in interest rates. Generally small
but matters for LEAPS.
Delta (Δ) — Price Sensitivity
The most important Greek — how your option moves with the stock
Delta measures the expected change in option price for a $1 move in the underlying. A 0.50 delta call gains ~$0.50
when the stock rises $1.
Moneyness Call Delta Put Delta Interpretation
Deep ITM 0.80 – 1.00 −0.80 – −1.00 Moves almost like the stock
ITM 0.60 – 0.80 −0.60 – −0.80 Strong directional exposure
ATM 0.45 – 0.55 −0.45 – −0.55 50/50 — maximum uncertainty
OTM 0.20 – 0.45 −0.20 – −0.45 Moderate probability of expiring ITM
Deep OTM 0.00 – 0.20 0.00 – −0.20 Low probability — lottery ticket territory DELTA AS PROBABILITY
Delta roughly approximates the probability of the option expiring ITM. A 0.30 delta call has roughly a 30% chance of finishing in the money.
This is why OTM sellers target low-delta strikes — they're betting on high probability of the option expiring worthless.
Gamma (Γ) — The Acceleration Factor
How fast delta changes — and why it matters near expiration
Gamma measures the rate of change of delta per $1 move in the underlying. If your option has a delta of 0.40 and gamma
of 0.05, a $1 move changes delta to 0.45. Gamma is the reason ATM options near expiry are so volatile.
LONG GAMMA (Buyer)
Your delta gets MORE favorable as the stock moves in your
direction. If you're long a call and the stock rises, your
delta increases — you make more on each subsequent $1
move.
Long gamma = convexity = you win more when you're right
than you lose when you're wrong.
Downside: you're paying theta for this privilege.
SHORT GAMMA (Seller)
Your delta gets LESS favorable as the stock moves against
you. If you're short a call and the stock rises, your delta
becomes more negative — you lose more on each
subsequent move.
Short gamma = concavity = losses accelerate.
Upside: you're collecting theta as compensation.
This is why gamma risk explodes near expiry.
Theta (Θ) — Time Decay
The constant drain on option value — your best friend or worst enemy
Theta measures the dollar amount an option loses per day from the passage of time alone. A theta of −$0.05 means the
option loses $5 per contract per day, all else equal.
Theta Decay Is Non-Linear
Theta accelerates as expiration approaches. An option with 60 DTE might lose $0.02/day. At 30 DTE, $0.05/day. At 7 DTE, $0.15/day. At 1
DTE, $0.40/day. The curve is roughly proportional to 1/√time.
ATM Options Decay Fastest
At-the-money options have the highest theta because they have the most extrinsic value. Deep ITM and deep OTM options have less extrinsic
value, so they decay more slowly.
The 45 DTE Sweet Spot
Income sellers typically enter at 30-45 DTE to capture the beginning of the acceleration curve. Close at 50% profit or 21 DTE — whichever
comes first. This maximizes theta capture per unit of risk.
Theta vs. Gamma Tradeoff
Long options pay theta to own gamma (convexity). Short options collect theta but are short gamma (concavity). This is the fundamental
tradeoff in options: time decay vs. acceleration.
Vega (V) — Volatility Sensitivity
How implied volatility changes affect your option's price
Vega measures the change in option price per 1 percentage point change in implied volatility. A vega of 0.10 means the
option gains $10 per contract if IV rises 1%.
LONG VEGA (Vol Buyer)
Profits when IV increases.
Long calls, long puts, long straddles and strangles are all
long vega.
Ideal before expected volatility events: earnings, FOMC,
geopolitical events.
Risk: IV crush after the event can destroy the position even
if direction is correct.
SHORT VEGA (Vol Seller)
Profits when IV decreases.
Short calls, short puts, iron condors, credit spreads are all
short vega.
Ideal when IV is elevated (high IV rank) and expected to
mean-revert.
Risk: unexpected vol spike inflates the options you sold —
losses can be sudden and large.
Second-Order Greeks & Portfolio Greeks
Advanced sensitivities and managing aggregate risk
Charm (Delta Decay) How delta changes with time. As expiration approaches, OTM deltas decrease and ITM
deltas increase. Important for delta-hedged portfolios.
Vanna How delta changes with IV. When IV rises, call deltas increase and put deltas decrease (for
OTM options). Critical during vol spikes.
Vomma (Volga) How vega changes with IV. Vega itself increases when IV rises. Explains why far OTM options
explode in value during market crashes.
Speed Rate of change of gamma. Third derivative of option price. Rarely used directly but explains
gamma behavior at extremes.
PORTFOLIO GREEKS Sum all position Greeks to get net portfolio delta, gamma, theta, and vega. A portfolio with net delta near
zero is market-neutral. Net theta positive = time is on your side. Monitor portfolio vega to know your vol
exposure.
How the Greeks Interact
The relationships that drive every P&L surprise
When This Happens... Delta Gamma Theta Vega

Stock falls $1 (call) − decreases unchanged* unchanged* unchanged*

The key insight: you're never trading just one Greek. Every options position is a bundle of exposures to price, time, volatility, and their
interactions. Understanding these relationships is what separates systematic options traders from gamblers.

Implied Volatility
The single most important concept in options trading
What Is Implied Volatility?
The market's forecast of future price movement — priced into every option
Implied volatility is the market's expectation of how much the underlying will move over the option's remaining life. It's
"implied" because we back-solve it from the option's market price using an options pricing model (Black-Scholes). It's not a
prediction of direction — only magnitude.
HISTORICAL VOLATILITY (HV)
What actually happened.
Calculated from past price data — the realized standard
deviation of returns over a lookback period (typically 20-30
days).
Backward-looking. Tells you how volatile the stock HAS
been.
Used as a baseline to judge whether IV is relatively high or
low.
IMPLIED VOLATILITY (IV)
What the market expects.
Back-solved from option prices. If options are expensive, IV
is high — the market expects big moves.
Forward-looking. Tells you what the market THINKS will
happen.
Drives strategy selection: sell premium when IV is high,
buy when IV is low.
IV Rank & IV Percentile
Measuring whether current IV is relatively high or low
IV RANK (IVR)
Where is current IV relative to its 52-week range?
Formula:
IVR = (Current IV − 52w Low) / (52w High − 52w Low)
IVR = 80% means current IV is near the top of its annual
range.
IVR > 50% = IV is elevated → favor selling premium
IVR < 30% = IV is depressed → favor buying premium
Simple and intuitive but can be skewed by a single extreme
spike.
IV PERCENTILE (IVP)
What percentage of days had LOWER IV than today?
Formula:
IVP = (Days with IV < Current IV) / Total Days
IVP = 85% means IV was lower than today 85% of the time.
IVP > 50% = more days were below current IV → selling
conditions
IVP < 30% = IV is unusually low → buying conditions
More robust than IVR because it's not skewed by outliers.
Volatility Smile & Skew
Why different strikes have different IVs — and what it tells you
If Black-Scholes were perfectly accurate, all strikes at the same expiration would have the same IV. They don't. The pattern
of IV across strikes reveals what the market is really pricing in.
Put Skew (Normal) OTM puts have higher IV than OTM calls. The market charges more for downside protection
because crashes happen faster and harder than rallies. This is the dominant pattern for equity
indices (SPY, QQQ). The steeper the skew, the more fearful the market.
Smile (Symmetric) Both OTM puts and OTM calls have elevated IV. Common in commodities and individual stocks
around earnings. The market expects a big move but isn't sure of direction.
Flat (Low Skew) IV is roughly equal across strikes. Unusual but occurs in low-vol environments when the market is
complacent. Can signal opportunity for vol buyers.
Skew as a Signal Rapidly steepening put skew often precedes corrections — institutions are buying protection.
Flattening skew after elevated levels can signal the worst is over. Monitor the 25-delta put/call IV
ratio.
VIX & Volatility Term Structure
The market's fear gauge and what the futures curve tells us
THE VIX
The CBOE Volatility Index measures expected 30-day S&P 500 volatility derived from SPX option prices. VIX < 15 = complacency,
15-20 = normal, 20-30 = elevated fear, > 30 = crisis mode. VIX itself is not directly tradeable — you trade VIX futures, VIX options,
or VIX ETPs (VXX, UVXY, SVXY).
CONTANGO (Normal)
Longer-dated VIX futures > shorter-dated. This is the
normal state (~80% of the time). The market expects more
uncertainty further out. Selling near-term premium and
buying longer-dated is the basis of calendar spread
strategies.
BACKWARDATION (Fear)
Shorter-dated VIX futures > longer-dated. Markets are
panicking NOW. Near-term fear is higher than long-term.
This is a warning signal. When backwardation persists, it
often means institutional hedging is extreme. Calendar
spreads break down.
Volatility Mean Reversion
The single most exploitable property of implied volatility
Volatility is mean-reverting. It spikes, then falls back. It compresses, then expands. This is the statistical edge
behind most options income strategies — sell premium when IV is elevated, buy it back after it contracts.
INCOME
Sell Premium When IV Is High
IVR > 50% or IVP > 60%. Options are expensive — you're getting paid more per unit
of risk. Iron condors, credit spreads, strangles, and covered calls all benefit from IV
contraction.
DIRECTIONAL / VOL
Buy Premium When IV Is Low
IVR < 30% or IVP < 30%. Options are cheap — you're paying less for potential big
moves. Long straddles, long strangles, and calendar spreads benefit from IV
expansion.
AVOID
Avoid Premium Sales in Low IV
Selling cheap options means low premium collected with the same max loss. Poor
risk/reward. If IV is already low, it can go lower but the contraction benefit is
minimal.
REGIME
HMM Connection
The HMM regime detector classifies volatility environments. Low-vol regimes →
favor selling. Transitional regimes → reduce size. High-vol regimes → selective
buying or sitting out.
IV-Driven Strategy Selection
Matching your strategy to the volatility environment
IV Environment IVR / IVP Best Strategies Avoid
Very High (> 80%) Spike / crisis Short strangles, iron condors, short puts Buying naked options
High (50-80%) Elevated Credit spreads, covered calls, jade lizards Long straddles, calendars
Normal (30-50%) Middle range Diagonals, butterflies, selective spreads Large undefined-risk
Low (15-30%) Compressed Long straddles, calendars, LEAPS Selling cheap premium
Very Low (< 15%) Extreme calm Long vol (straddles), ratio backspreads Any premium sale
This is the foundation of systematic options trading: identify the IV environment, select the strategy that exploits mean reversion, size
according to the regime, and manage risk. Everything else is refinement.

Income Strategies
Selling premium systematically — the foundation of options income
Cash-Secured Puts
Selling the right to sell you shares at a price you'd happily pay
CASH-SECURED PUT
Sell an OTM put and hold enough cash to buy 100 shares if assigned. You're getting paid to wait for a lower price.
Max Profit Premium collected
Max Loss Strike × 100 − premium (own
shares at strike)
Breakeven Strike − premium per share
Ideal IV High (IVR > 50%)
DTE 30-45 days
Delta −0.20 to −0.30

→ Close at 50% profit (mechanical, no discretion)
→ If tested (price drops to strike), roll down and out
→ If assigned, transition to covered calls (the wheel)
→ Never sell puts on stocks you wouldn't own
→ Size: 1 contract per $5k–$10k account value

Covered Calls
Generating income on shares you already own
COVERED CALL
Own 100 shares + sell 1 OTM call against them. You're capping upside in exchange for income. The shares are
your "cover" — no margin required.
Max Profit Premium + (strike − cost basis) ×
100
Max Loss Cost basis × 100 − premium (stock

Breakeven Cost basis − premium per share

→ If stock rallies past strike, decide: let assign or roll up
→ If stock drops significantly, buy back cheap call, wait
→ Sell calls ABOVE your cost basis (avoid selling at a loss)
→ Monthly cycle: sell → manage → expire/close → repeat
→ In bull regime: higher strikes (0.20 delta, more upside)
The Wheel Strategy
The complete income cycle: sell puts → get assigned → sell calls → repeat
1
Sell Cash-Secured Put
Sell an OTM put at a strike where you'd be happy to own shares. Collect premium. If price stays above strike, put expires
worthless — keep premium and repeat.
2
Get Assigned (Buy Shares)
If price drops below strike at expiration, you're assigned: you buy 100 shares at the strike price. Your effective cost basis = strike −
premium collected.
3
Sell Covered Call
Now sell an OTM call above your cost basis against the shares you own. Collect more premium. If price stays below call strike, call
expires — keep premium and repeat.
4
Get Called Away (Sell Shares)
If price rises above call strike, shares get called away — you sell at the strike price + keep the premium. Profit = call strike − cost
basis + all premiums. Restart at step 1. The wheel works best on stable, dividend-paying stocks you'd want to own anyway. It underperforms in strong bull markets (capped upside) and
strong bear markets (holding declining shares). Best in choppy/neutral regimes — exactly where the HMM can gate it.
Iron Condor
Betting the stock stays in a range — defined risk on both sides
IRON CONDOR
Sell OTM put spread + sell OTM call spread = collect premium from both sides. You're betting the stock stays
between your short strikes.
STRUCTURE
Buy 1 OTM put (lower strike — protection)
Sell 1 OTM put (higher — income)
Sell 1 OTM call (lower — income)
Buy 1 OTM call (higher strike — protection)
Max Profit Net premium collected
Max Loss Width of widest spread − premium
Breakeven Short strikes ± premium
Ideal IV High (IVR > 50%)
DTE 30-45 days
Delta ~0 (market neutral)
MANAGEMENT
→ Close at 50% of max profit
→ If one side is tested, consider rolling
→ Close both sides at 21 DTE if not already profitable
→ Never let it go to expiration — gamma risk
→ Standard widths: $1-$5 (depends on underlying)
→ Short strikes at ~0.16 delta (1 SD) each side
→ Expected win rate: ~70-80% over large sample
Credit Spreads
Directionally biased, defined-risk premium selling
BULL PUT SPREAD (Bullish)
Sell higher-strike put + buy lower-strike put.
Collect net credit. Stock stays above short put = full profit.
Max profit: credit received
Max loss: width − credit
Breakeven: short strike − credit
Use when: bullish or neutral, high IV
Short strike: 0.20-0.30 delta
DTE: 30-45 days
Width: $1-$5 depending on risk tolerance
Management: close at 50% profit, roll if tested, never hold to
expiry.
BEAR CALL SPREAD (Bearish)
Sell lower-strike call + buy higher-strike call.
Collect net credit. Stock stays below short call = full profit.
Max profit: credit received
Max loss: width − credit
Breakeven: short strike + credit
Use when: bearish or neutral, high IV
Short strike: 0.20-0.30 delta
DTE: 30-45 days
Width: $1-$5 depending on risk tolerance
Management: close at 50% profit, roll if tested, never hold to
expiry.
Short Strangle
Selling both sides naked — highest premium, highest risk
SHORT STRANGLE
Sell OTM put + sell OTM call (no protection legs). Maximum premium collected but undefined risk on both sides.
Requires margin and a strong stomach.
Max Profit Total premium collected
Max Loss Unlimited (naked on both sides)
Breakeven Short strikes ± total premium
Ideal IV Very high (IVR > 60%)
DTE 45 days
Delta ~0.16 each side (~1 standard
deviation)
MANAGEMENT & RISK
→ Close at 50% profit — don't wait for full decay
→ Roll tested side to same DTE at next strike out
→ If stock blows through one side, close and reassess
→ Requires Level 4+ options approval
→ Margin requirement: typically 20% of underlying
→ Size: no more than 2-5% of portfolio per strangle
→ ONLY in high IV environments — not worth the risk in low IV
Advanced Income: Jade Lizard & Iron Butterfly
Variations that modify the risk/reward profile
JADE LIZARD
Short put + short call spread (bear call spread). No upside risk
if structured correctly.
The call spread premium + put premium must exceed the call
spread width. This eliminates upside risk entirely.
Downside risk remains: if stock crashes through the short put,
losses can be large.
Best when: bullish-neutral bias, want to eliminate one side of
risk entirely.
Used as an alternative to iron condors when you want more
premium on the put side and are willing to accept directional
downside.
IRON BUTTERFLY
Sell ATM put + sell ATM call + buy OTM put + buy OTM call. All
at or near the same strike for the shorts.
Higher premium than iron condor (ATM strikes have max
extrinsic) but narrower profit zone.
Max profit: total premium collected
Max loss: wing width − premium
Breakeven: ATM strike ± premium
Best when: expecting very low movement, pinning action, low
realized vol.
Win rate is lower than iron condor (~40-50%) but profit per
winner is larger. Risk/reward tradeoff.
Rolling & Adjustment Framework
How to manage income trades that move against you
What Is Rolling? Closing your current position and opening a new one at a different strike, expiration, or both.
You're buying time and/or adjusting your range. Ideally done for a net credit — never roll for
a debit.
Roll Out (Same Strike, Later Date) Extends time for the stock to move back in your favor. Works when the setup is still valid but
you need more time. Most common adjustment for tested puts.
Roll Down/Up (Different Strike,
Same Date) Moves your strike to a more favorable level. Reduces premium collected but improves your
odds. Use when the original thesis has weakened.
Roll Out and Down/Up Combines both. The most flexible adjustment. Adds time AND improves strikes. Must be
done for a net credit to be worthwhile.
When NOT to Roll If the trade is a loser due to a fundamental change in the thesis (not just price movement),
take the loss. Rolling a bad trade just extends the pain. If you can't roll for a credit, close it.
The 21 DTE Rule If a trade hasn't worked by 21 DTE, close it regardless. Gamma risk is too high. Take the loss,
move on, and re-enter next cycle if the setup is still valid.

Directional Strategies
Capturing moves with defined risk and leverage
Long Calls & Long Puts
The simplest directional bet — buy an option, ride the move
LONG CALL (Bullish)
Buy a call when you expect the stock to rise. Defined risk
(premium paid), unlimited upside.
Strike selection:
• ITM (0.70 delta): stock replacement, expensive
• ATM (0.50 delta): balanced risk/reward
• OTM (0.30 delta): cheap, needs big move
DTE: 45-90 days minimum (fight theta)
IV: buy when IV is low (IVR < 30%)
Management: set profit target (50-100%), cut losses at 50%
of premium, never hold past 21 DTE.
LONG PUT (Bearish)
Buy a put when you expect the stock to fall. Defined risk
(premium paid), profit down to zero.
Strike selection:
• ITM (−0.70 delta): most conviction, expensive
• ATM (−0.50 delta): balanced
• OTM (−0.30 delta): cheap hedge or spec
DTE: 45-90 days minimum
IV: buy when IV is low — puts get expensive fast in selloffs
Management: same rules as calls. Puts have a natural
advantage: stocks fall faster than they rise (volatility spikes
help you).
Debit Spreads
Defined-risk directional bets with reduced cost
BULL CALL SPREAD
Buy lower-strike call + sell higher-strike call.
Pay net debit. Cheaper than naked call but caps upside at the
short strike.
Max profit: width − debit
Max loss: debit paid
Breakeven: long strike + debit
Ideal when: moderately bullish, IV is elevated (selling the
short leg offsets high IV cost)
DTE: 30-60 days
Width: $2-$5 for most underlyings
Long strike: ATM or slightly ITM
Short strike: OTM at target price
BEAR PUT SPREAD
Buy higher-strike put + sell lower-strike put.
Pay net debit. Cheaper than naked put, caps profit at short
strike.
Max profit: width − debit
Max loss: debit paid
Breakeven: long strike − debit
Ideal when: moderately bearish, IV is elevated
DTE: 30-60 days
Width: $2-$5
Long strike: ATM or slightly ITM
Short strike: OTM at downside target
Management: close at 50-75% of max profit. Cut at 50% loss
of debit.
Directional Momentum Plays
Breakout entries with options for leverage and defined risk
Using options for momentum/breakout entries gives you leverage with defined risk. Instead of buying shares on a QQQ
trend break, buy a debit spread or long call. The key is timing, strike selection, and expiration.
Opening Range Breakout Enter long calls (bullish) or puts (bearish) when the underlying breaks above/below the first 15-30
min range. Use ATM or slightly OTM options. 0DTE or 1DTE for pure momentum. Tight stops —
close at 50% loss of premium.
Trend Line Break When a major trend line breaks with volume confirmation, enter debit spreads in the direction of
the break. 30-45 DTE to avoid theta bleed. ATM long strike, $3-$5 wide.
Power Hour (3-4 PM ET) Institutional flows create momentum in the last hour. Enter only with clear directional bias
confirmed by volume. 0DTE or 1DTE options. Small size — these are high-gamma, high-risk plays.
Earnings Momentum Buy debit spreads 1-2 weeks before earnings when IV is rising (pre-event IV expansion). Close
before the event or hold through with defined risk. IV crush after earnings destroys long premium.
Regime-Gated Entries Only take momentum entries when the HMM confirms a trending regime. Choppy/neutral regimes
eat momentum plays alive. Bull regime = long bias, bear regime = short bias.
LEAPS — Long-Term Equity Anticipation
Securities
Stock replacement with leverage and defined risk
LEAPS are options with more than 1 year to expiration. Used as stock replacement: buy a deep ITM call instead of 100
shares. You control the same upside with a fraction of the capital and defined downside risk.
Stock Replacement Buy deep ITM call (0.80+ delta), 12-24 months out. Controls 100 shares for 15-25% of the
cost. Delta near 1.0 means it moves almost dollar-for-dollar with the stock. Minimal theta
decay this far out.
Poor Man's Covered Call Buy a LEAP call (long-term) + sell short-term OTM calls against it monthly. Same concept as
covered calls but with less capital. The LEAP is your "stock" position. Collect monthly
premium.
Sizing Advantage With $50,000, you could buy 100 shares of a $500 stock ($50k) OR 1 LEAP for ~$7,500. The
remaining $42,500 stays in cash or other positions. Same upside exposure, 85% less capital at
risk.
Risks IV contraction can hurt (you paid for time value). If stock drops below LEAP strike, you lose
the entire premium. No dividends. Roll or close at 6 months remaining — don't let theta
accelerate.

Volatility Strategies
Trading the magnitude of moves — not the direction
Long Straddle & Long Strangle
Betting on a big move in either direction
LONG STRADDLE
Buy ATM call + buy ATM put at the same strike.
Pay debit for both. Profit if stock moves far enough in either
direction to overcome the cost of both premiums.
Max profit: unlimited (either direction)
Max loss: total debit (both premiums)
Breakeven: strike ± total debit
Ideal: low IV (cheap entry), expecting a big move
DTE: 30-60 days
Best before: earnings, FOMC, major catalysts
Risk: IV crush + theta if stock doesn't move
LONG STRANGLE
Buy OTM call + buy OTM put (different strikes).
Cheaper than straddle but needs a bigger move to profit.
Both options start OTM.
Max profit: unlimited
Max loss: total debit
Breakeven: put strike − debit / call strike + debit
Ideal: very low IV, expecting explosive move
DTE: 45-90 days (more time for the move)
Management: close one side at 50%+ profit if stock trends.
Close both at 21 DTE if no move.
Regime gate: only in low-vol regimes expecting transition.
Calendar Spreads
Exploiting the differential decay between near and far expirations
CALENDAR SPREAD (Time Spread)
Sell near-term option + buy same-strike longer-term option. You're selling fast-decaying theta and owning slowdecaying theta. Net debit trade.
Max Profit Difference in decay rates at expiry
Max Loss Debit paid
Ideal Scenario Stock pins near the strike at frontmonth expiry
Ideal IV Low to normal (benefits from IV
expansion)
Structure Sell 30 DTE, buy 60-90 DTE, same
strike
Delta ~0 (neutral)
WHY CALENDARS WORK
→ Near-term option decays faster than far-term
→ If stock stays near strike, front option dies → profit
→ Back-month option retains most of its value
→ Positive vega: benefits from IV expansion
→ Works best in contango (normal VIX term structure)
→ Breaks down in backwardation (crisis) — avoid
→ Close front leg before expiry, sell another, or close all
Diagonal Spreads
Calendar spreads with a directional bias
A diagonal spread is a calendar spread where the strikes are different. This adds a directional component — you can be long
vega and theta while also having a bullish or bearish bias.
BULLISH DIAGONAL (Call)
Buy ITM call (60-90 DTE)
Sell OTM call (30 DTE)
The LEAPS-like long call appreciates as stock rises. The
short call generates income. Repeat monthly.
This IS the Poor Man's Covered Call — a capital-efficient
covered call alternative.
Max loss: debit paid
Ideal: moderate bullish bias + income
BEARISH DIAGONAL (Put)
Buy ITM put (60-90 DTE)
Sell OTM put (30 DTE)
Mirror of the bullish version for bearish bias. Less common
but useful in bear regimes.
The short put generates income while the long put
provides directional exposure.
Max loss: debit paid
Ideal: moderate bearish bias + income
Earnings Plays
Trading the volatility event cycle around quarterly reports
Pre-Earnings IV Expansion IV rises in the 1-2 weeks before earnings as uncertainty increases. Buy calendars or straddles
early to capture this expansion. Don't hold through earnings unless you have a directional
view.
IV Crush (Post-Earnings) IV collapses immediately after the announcement — often 30-60% overnight. This is why
selling premium before earnings is tempting but dangerous: direction can overwhelm the IV
crush benefit.
Earnings Straddle Sell Sell ATM straddle the day before earnings. Profit from IV crush if the stock doesn't move
beyond breakevens. High win rate but catastrophic when wrong — unlimited risk.
Defined-Risk Earnings Play Sell iron condors or iron butterflies before earnings instead of naked straddles. Cap your risk.
The premium is lower but you survive the outlier events that blow up undefined-risk trades.
Historical Move Analysis Compare implied move (from ATM straddle price) to historical earnings moves. If implied >
historical average, favor selling. If implied < historical, favor buying. Data available from
options analytics tools.

Hedging & Portfolio Overlay
Using options to protect your portfolio — not just speculate
Protective Puts & Collars
Insurance for your stock positions
PROTECTIVE PUT
Own 100 shares + buy 1 OTM put.
Floor your downside at the put strike. Unlimited upside
remains. The put is your insurance policy.
Cost: premium paid (reduces total return)
Max loss: stock price − put strike + premium
Breakeven: stock price + premium
Best when: high conviction long but want crash protection
Strike: 5-10% below current price
DTE: 60-90 days or quarterly
Expensive as ongoing insurance — most effective when IV is
LOW (cheap puts).
COLLAR
Own 100 shares + buy OTM put + sell OTM call.
The sold call funds the put — can be zero-cost. You cap
upside but guarantee a floor.
Max profit: call strike − stock price
Max loss: stock price − put strike
Cost: often net zero or small credit
Best when: protecting gains, market uncertainty
Put: 5-10% below current price
Call: 5-10% above current price
DTE: 60-90 days
The HMM can trigger collars automatically when regime shifts
to high-vol.
Tail Risk Hedging
Protection against black swan events — when everything else fails
What Is Tail Risk? Events in the statistical tails — 3+ standard deviation moves. COVID crash (−34% in 23 days), 2008
GFC, flash crashes. Normal risk management doesn't protect against these because they're outside
the model's assumptions.
Deep OTM Put Strategy Buy 5-10% OTM puts, 30-60 DTE, on SPY or your most concentrated position. Allocate 0.5-1% of
portfolio per month. They expire worthless most months — but when they hit, they can return 10-
50x.
VIX Call Hedging Buy OTM VIX calls (VIX 25-30 strike) as portfolio insurance. VIX tends to spike when equities crash.
VIX calls can increase 5-20x during panics. Cost: small premium, limited risk.
Put Ratio Backspread Buy 2 OTM puts, sell 1 ATM put. Net zero or small credit. Limited risk near current price,
accelerating gains on a crash. The extra long put provides convexity in the tail.
Regime-Triggered Hedging The HMM can automate this: when the regime shifts to high-vol or crash mode, automatically buy
protective puts or VIX calls. In low-vol regimes, let the hedges expire and don't replace them.
Portfolio-Level Options Overlay
Systematic hedging and income generation on your entire portfolio
Delta-Neutral Portfolio Use options to bring net portfolio delta close to zero. Long stocks + short delta via puts or
call sells. Profit from theta and vol trading while being market-neutral.
Systematic Covered Calls Sell covered calls on every position monthly. Reduce cost basis over time. In bull regimes,
use higher strikes (0.20 delta). In neutral, use closer strikes (0.30 delta). In bear, don't sell
calls.
Rolling Protective Puts Maintain continuous put protection on concentrated positions. Roll quarterly. In low-vol
regimes (cheap puts), buy longer-dated protection. In high-vol (expensive puts), use
collars.
Gamma Scalping Maintain a long-gamma portfolio position (long options). As the stock moves, delta
changes — buy/sell shares to re-hedge. Captures realized vol vs implied vol spread.
Advanced, requires active management.

Options Risk Management
Position sizing, portfolio Greeks, and managing the undefined
Defined vs. Undefined Risk
The most important distinction in options trading
DEFINED RISK
You know your max loss before entering.
Examples: debit spreads, credit spreads, iron condors, iron
butterflies, long options, LEAPS.
Max loss = debit paid (long) or width − credit (spreads).
No margin surprise. No assignment risk beyond the defined
amount.
Downside: lower premium collected (for sellers), wider
breakevens.
Recommended for: accounts under $50k, newer traders,
anyone who wants to sleep at night.
UNDEFINED RISK
Your max loss is theoretically unlimited (or very large).
Examples: naked puts, naked calls, short strangles, short
straddles.
Max loss: stock goes to zero (puts) or infinity (calls).
Requires significant margin. Assignment risk is real. One
outlier can erase a year of profits.
Upside: higher premium, better win rates, more flexible
management.
Recommended for: experienced traders, larger accounts
($100k+), with strict risk rules.
Position Sizing for Options
How much capital to allocate per trade
The 1-5% Rule Never risk more than 1-5% of your portfolio on a single options trade. For defined-risk trades,
this means the max loss should be 1-5% of equity. For undefined-risk, use notional exposure.
Contract Sizing: Defined Risk Max contracts = (Portfolio × Risk %) / Max loss per contract. Example: $100k portfolio, 2% risk
= $2,000 max loss. If an iron condor has $500 max loss, trade 4 contracts maximum.
Contract Sizing: Undefined Risk Size by notional exposure. If selling a naked put at $500 strike, your notional is $50,000 (500 ×
100). Keep total naked notional under 50% of portfolio. In practice, 1-2 contracts for most
accounts.
Portfolio Allocation Total options allocation: 20-50% of portfolio. Cash reserve: always keep 50%+ in cash/bonds
for margin requirements and opportunities. Never be fully deployed in options.
Scaling by IV Environment In high IV: smaller positions (more premium per contract, same dollar risk with fewer
contracts). In low IV: larger positions allowed since max loss per contract is lower.
Managing Portfolio Greeks
Monitoring aggregate risk across all your options positions
Greek What to Monitor Target Range Action if Exceeded
Net Delta Directional exposure ±20% of portfolio value Hedge with opposite-delta trades
Net Gamma Acceleration risk Positive or near zero Close short-gamma near expiry
Net Theta Daily income/cost +$X to +$3X daily Reduce if theta too negative
Net Vega Vol exposure Regime-dependent In high IV: negative. In low IV:
positive
Beta-Weighted Delta SPY-equivalent exposure ±30% of portfolio Hedge with SPY options
Beta-weighted delta converts all positions to SPY-equivalent exposure. If your beta-weighted delta is +50, you're effectively long 50
shares of SPY. This is the single most useful portfolio risk metric for options traders.
When to Take a Loss
The hardest skill in options — knowing when to exit a loser
Mechanical Exit Rules Set rules BEFORE entering: "If this trade loses 2× the credit received, I close it." No discretion. This
prevents hope from turning a small loss into a catastrophic one.
Time-Based Exits If a trade hasn't worked by 21 DTE, close it. Gamma risk is too high to hold. The remaining
premium isn't worth the tail risk. Re-enter next cycle if setup is still valid.
Thesis Violation If the reason you entered no longer exists (earnings surprise, regime change, sector rotation), close
immediately regardless of P&L. The trade is no longer what you signed up for.
Portfolio-Level Stops If total portfolio drawdown exceeds 5-10% in a week, close all positions and reassess. Individual
trade management isn't enough when correlations spike. The regime trader's kill switch handles
this.
Never Average Down Adding to a losing undefined-risk position is how accounts blow up. If a short strangle is breached,
close it. Don't sell more premium "to improve the breakeven." The market doesn't care about your
breakeven.
Rolling vs. Closing Roll only if: (1) the thesis is intact, (2) you can roll for a credit, and (3) the new position has a
positive expected value on its own merits. If any of these fail, close the trade.

Calculating Greeks Yourself
When to pull from an API vs. computing your own
Black-Scholes Model The classic. Inputs: stock price, strike, time to expiry, risk-free rate, IV. Outputs: call/put price,
delta, gamma, theta, vega, rho. Works for European options. Good enough for daily-level
analysis on American options.
Binomial Model More accurate for American options (early exercise). Tree-based approach. Slower to
compute but handles dividends and early exercise correctly. Use for dividend-paying stocks
where early assignment matters.
When to Calculate vs. Pull Calculate your own when: you need Greeks for backtesting (no API for historical), you want to
avoid API costs, or you need Greeks at non-standard parameters. Pull from API when: speed
matters, real-time trading, or official prices.
IV Back-Solving Given a market price, back-solve for IV using Newton's method or bisection. This is how IV is
calculated — it's the volatility that makes Black-Scholes output match the market price.
Libraries: py_vollib, mibian, QuantLib.

Order Types Limit orders only for options (never market — spreads are too wide). Multi-leg orders for spreads
(single ticket, better fills). GTC or day orders. Set price at mid or slightly above mid for fills.
Execution
Considerations

(spreads are widest).

slippage is larger, fills are slower, and assignment mechanics need to be tested.

Gamma Levels & Dealer Positioning
How market maker hedging creates support and resistance
What Are Gamma Levels? Market makers who sell options must hedge their delta exposure. The aggregate gamma
exposure across all outstanding options creates zones where market makers either amplify or
dampen price moves.
Positive Gamma (Dampening) When dealers are long gamma (net bought options at a strike), they buy dips and sell rallies
to maintain delta neutrality. This dampens volatility and creates pinning action around highOI strikes.
Negative Gamma (Amplifying) When dealers are short gamma (net sold options at a strike), they sell into dips and buy into
rallies — amplifying moves. This creates the explosive moves and breakouts that momentum
traders look for.
Zero Gamma Line (GEX Flip) The price level where dealer gamma flips from positive to negative. Above this line, dealers
dampen moves (supportive). Below it, dealers amplify moves (destabilizing). This level acts as
a key support/resistance.
Using Gamma Levels Monitor the zero-gamma line and high-gamma strikes for your underlying. Sell premium near
high-positive-gamma levels (pinning). Buy premium or avoid selling near negative-gamma
zones. Tools: SpotGamma, Unusual Whales, GEX data.

CORE MODULE — chain_fetcher.py:

Class: OptionsChainFetcher

init(self, cache_dir="./cache", cache_ttl_minutes=15):

- Initialize with local file cache. Default 15-minute TTL for cached chain data.

- yfinance has no API key requirement but is rate-limited. Cache aggressively.

Method: get_chain(self, symbol: str, expiration: str = None) -> ChainData:

- If expiration is None, return the nearest monthly expiration (not weekly — filter for standard monthly expirations which fall on the third Friday).

- Pull the full options chain for the given symbol and expiration using yfinance.

- yfinance returns calls and puts as separate DataFrames. Merge them into a single structured output.

- For each contract, extract and clean: contractSymbol, strike, lastPrice, bid, ask, volume, openInterest, impliedVolatility, lastTradeDate, inTheMoney.

- Calculate the mid price as (bid + ask) / 2. If bid or ask is zero or NaN, use lastPrice as fallback.

- Calculate the bid-ask spread as (ask - bid). Flag contracts where spread > 20% of mid price as "wide spread — low liquidity."

- Add a "moneyness" column: ITM, ATM (within 0.5% of current price), or OTM.

- Add a "delta_estimate" column: use the Black-Scholes delta formula with the contract's implied volatility, current stock price, strike, time to expiry, and risk-free rate (pull the current 3-month Treasury rate from yfinance or hardcode 5.0% as default).

- Sort calls by strike ascending, puts by strike ascending.

- Return a ChainData object (defined in chain_models.py).

Method: get_all_expirations(self, symbol: str) -> List[str]:

- Return all available expiration dates for the symbol.

- Add metadata: days to expiration (DTE) for each, and flag whether it's weekly or monthly.

- Monthly expirations are the third Friday of each month. Everything else is weekly.

Method: get_multi_expiry_chain(self, symbol: str, dte_range: Tuple[int, int] = (7, 90)) -> Dict[str, ChainData]:

- Pull chains for all expirations within the DTE range.

- Return a dict keyed by expiration date string.

Method: get_underlying_price(self, symbol: str) -> UnderlyingInfo:

- Return current price, 52-week high/low, average volume, market cap.

- Calculate 20-day and 50-day historical volatility from daily returns.

- Return as an UnderlyingInfo dataclass.

CORE — black_scholes.py:

All Black-Scholes European pricing. Inputs: S (stock), K (strike), T (time in years), r (rate), sigma (vol), q (dividend yield, default 0).

Helper: d1 = (ln(S/K) + (r-q+sigma^2/2)*T) / (sigma*sqrt(T)), d2 = d1 - sigma*sqrt(T)

Functions:

- call_price(S,K,T,r,sigma,q=0) = S*exp(-qT)*N(d1) - K*exp(-rT)*N(d2)

- put_price(S,K,T,r,sigma,q=0) = K*exp(-rT)*N(-d2) - S*exp(-qT)*N(-d1)

- delta: Call = exp(-qT)*N(d1) [0 to 1], Put = exp(-qT)*(N(d1)-1) [-1 to 0]

- gamma = exp(-qT)*n(d1) / (S*sigma*sqrt(T)). Same for calls and puts.

- theta: Call = -(S*sigma*exp(-qT)*n(d1))/(2*sqrt(T)) - r*K*exp(-rT)*N(d2) + q*S*exp(-qT)*N(d1). Put version with flipped signs. Return daily (divide by 365).

- vega = S*exp(-qT)*sqrt(T)*n(d1). Same for calls/puts. Return per 1% IV (divide by 100).

- rho: Call = K*T*exp(-rT)*N(d2). Put = -K*T*exp(-rT)*N(-d2). Per 1% rate change.

- all_greeks(S,K,T,r,sigma,type,q=0) -> dict with price, delta, gamma, theta, vega, rho, d1, d2.

Second-order:

- charm: rate of delta change with time

- vanna: rate of delta change with IV

- vomma: rate of vega change with IV

Edge cases: T<=0 return intrinsic, T<0.001 use 0.001, sigma<=0 raise error.

IV SOLVER — iv_solver.py:

implied_volatility(market_price, S, K, T, r, type, q=0, precision=1e-6, max_iter=100):

- Newton-Raphson: sigma_new = sigma_old - (bs_price - market_price) / vega. Initial guess via Brenner-Subrahmanyam.

- Fallback to Brent's method (scipy.optimize.brentq, bounds [0.001, 5.0]) if Newton doesn't converge.

- Return NaN if market price < intrinsic or <= 0.

iv_surface(chains_data, S, r) -> DataFrame with strike, expiration, dte, type, market_price, iv, delta.

Class: IVAnalyzer

Method: historical_volatility(symbol, window=20) -> pd.Series:

- Pull 1 year of daily close prices using yfinance.

- Calculate log returns: ln(close_t / close_t-1).

- Rolling std dev of returns over window days.

- Annualize: multiply by sqrt(252).

- Return as a time series. Calculate for windows: 10, 20, 30, 60 days.

Method: current_iv(symbol) -> float:

- Pull the nearest monthly ATM option chain.

- Find the two strikes closest to the current price (one above, one below).

- Linear interpolate their IVs to estimate true ATM IV.

- Average call and put ATM IV for a cleaner estimate.

- Get current ATM IV.

- Same approximation as iv_rank but percentile-based.

- Calculate IVR and IVP.

- Classify: "very_low" (<15%), "low" (15-30%), "normal" (30-50%), "high" (50-80%), "very_high" (>80%).

- Return IVEnvironment dataclass with all metrics and classification.

Method: hv_iv_spread(symbol) -> dict:

- Current IV minus current HV20.

- Positive spread = IV elevated relative to realized (options are expensive).

- Negative spread = IV depressed (options are cheap).

- Historical average spread for context.

Method: vol_cone(symbol, windows=[10,20,30,60,90]) -> pd.DataFrame:

- For each window, calculate the min, 25th percentile, median, 75th percentile, and max of historical vol over the past year.

- Plot as a "volatility cone" — shows where current vol sits relative to historical norms at each timeframe.

- CALL_OPEN: Own shares + short call. Monitoring. → Transition to SCANNING if called away, or stay if call expires worthless (sell another).

- Log entry to trade journal.

Method: manage_csp(trade: WheelTrade, current_chain) -> WheelAction:

- CHECK 1: Is current P&L >= 50% of max profit? → Action: CLOSE (take profit).

- CHECK 2: Is DTE <= 21? → Action: CLOSE (gamma risk too high, re-enter next cycle).

- CHECK 3: Is the put ITM (price below strike)? → Evaluate rolling:

  - Can we roll down and out for a net credit? If yes → Action: ROLL with new contract details.

  - If no credit available → Action: HOLD (accept assignment if it comes).

- CHECK 4: Has IV environment changed dramatically (IVR dropped below 20%)? → Action: CLOSE (edge is gone).

- CHECK 5: Is there an earnings date within 7 days? → Action: CLOSE (avoid earnings risk).

- DEFAULT: HOLD.

- Return WheelAction with action type, reasoning, and new contract details if rolling.

Method: handle_assignment(trade: WheelTrade) -> WheelTrade:

- Record: shares acquired at strike price. Effective cost basis = strike - premium_collected.

- Transition state to CALL_OPEN.

- Immediately scan for covered call: strike ABOVE cost basis, 0.20-0.30 delta, 30-45 DTE.

- Return updated WheelTrade with CC details.

Method: enter_cc(symbol, shares, cost_basis, account_equity) -> WheelTrade:

- If no strikes above cost basis have decent premium (< $0.30), widen DTE to 45-60.

- Calculate: premium, breakeven (adjusted cost basis), max profit if called away, annualized return.

Method: manage_cc(trade: WheelTrade, current_chain) -> WheelAction:

- CHECK 1: P&L >= 50% of max profit? → CLOSE call, sell another next cycle.

- CHECK 2: DTE <= 21? → CLOSE, sell new call.

- CHECK 3: Stock rallied past strike? → Evaluate: let assign (profit + premium) or roll up and out.

- CHECK 4: Stock dropped significantly (>10% from entry)? → CLOSE call (it's cheap), hold shares, wait for recovery before selling another.

--- data/chain_fetcher.py ---

Class: OptionsChainFetcher

Method: get_chain(self, symbol: str, expiration: str = None) -> dict:

- If expiration is None, return the nearest monthly expiration (third Friday of the month, not weekly).

- Pull full options chain using yfinance: ticker = yf.Ticker(symbol), chain = ticker.option_chain(expiration).

Max pain calculation: For each strike in the chain, calculate the total dollar value all outstanding calls and puts would lose if the stock expired at that strike. Sum call losses + put losses at each strike. The strike where this total is maximized is max pain.

--- engine/black_scholes.py ---

All functions use standard Black-Scholes for European options.

Inputs: S (stock price), K (strike), T (time in years, e.g. 30/365), r (risk-free rate, e.g. 0.05), sigma (volatility, e.g. 0.25), q (dividend yield, default 0.0).

Helpers: d1 = (ln(S/K) + (r - q + sigma²/2)  T) / (sigma  sqrt(T)), d2 = d1 - sigma * sqrt(T).

Functions:

- call_price(S, K, T, r, sigma, q=0.0): C = S*exp(-qT)*N(d1) - K*exp(-rT)*N(d2)

- put_price(S, K, T, r, sigma, q=0.0): P = K*exp(-rT)*N(-d2) - S*exp(-qT)*N(-d1)

- delta(S, K, T, r, sigma, option_type, q=0.0): Call = exp(-qT)*N(d1). Put = exp(-qT)*(N(d1)-1).

- gamma(S, K, T, r, sigma, q=0.0): = exp(-qT)*n(d1) / (S*sigma*sqrt(T)). Same for calls/puts.

- theta(S, K, T, r, sigma, option_type, q=0.0): Standard BS theta formula. Return daily (÷365). Negative for long options.

- vega(S, K, T, r, sigma, q=0.0): = S*exp(-qT)*sqrt(T)*n(d1). Return per 1% IV change (÷100).

- rho(S, K, T, r, sigma, option_type, q=0.0): Call = K*T*exp(-rT)*N(d2). Put = -K*T*exp(-rT)*N(-d2). Per 1% (÷100).

- all_greeks(S, K, T, r, sigma, option_type, q=0.0) -> dict: price, delta, gamma, theta_daily, vega_pct, rho_pct.

Edge cases: T<=0 return intrinsic value, delta 0 or 1, other Greeks 0. T<0.001 use 0.001. sigma<=0 raise error.

N() = scipy.stats.norm.cdf, n() = scipy.stats.norm.pdf.

--- engine/iv_solver.py ---

Function: implied_volatility(market_price, S, K, T, r, option_type, q=0.0, precision=1e-6, max_iter=100):

- Newton-Raphson: initial guess via Brenner-Subrahmanyam (sigma_0 = sqrt(2*pi/T) * market_price / S).

- Iterate: sigma_new = sigma_old - (bs_price(sigma_old) - market_price) / vega(sigma_old).

- Stop when |bs_price - market_price| < precision or max_iter.

- If Newton fails (vega near zero), fall back to scipy.optimize.brentq with bounds [0.001, 5.0].

- Return NaN if market_price < intrinsic value or <= 0.

--- strategy/iron_condor.py ---

Class: IronCondorStrategy

Method: scan_candidates(self, universe: list, iv_environment_filter: str = "high") -> list:

- Default universe: ["SPY", "QQQ", "IWM", "DIA", "AAPL", "MSFT", "GOOGL", "AMZN", "NVDA", "META", "JPM", "V", "XLF", "XLK", "XLE", "XLV", "GLD", "TLT"].

- For each ticker:

  1. Get IV environment. Filter: only tickers where IVR > 50% (configurable).

  2. Check earnings: pull next earnings date from yfinance. Skip if earnings within 14 days.

  3. Check liquidity: average daily option volume > 1000 contracts. Average OI > 500.

  4. Check bid-ask: ATM bid-ask spread < 10% of mid price.

- For each passing ticker, structure a sample iron condor:

  - Short put: closest strike to 0.16 delta (approximately 1 standard deviation OTM).

  - Long put: short_put_strike - width (default $5, configurable).

  - Short call: closest strike to 0.16 delta on call side.

  - Long call: short_call_strike + width.

  - Expiration: nearest monthly with 30-45 DTE.

- Calculate for each candidate:

  - Net credit = short_put_mid + short_call_mid - long_put_mid - long_call_mid (all mids).

  - Max profit = net_credit × 100.

  - Max loss = (wider_wing_width - net_credit) × 100.

  - Breakeven_lower = short_put_strike - net_credit.

  - Breakeven_upper = short_call_strike + net_credit.

  - Probability of profit = 1 - abs(delta_short_put) - abs(delta_short_call). Approximate.

  - Annualized return = (net_credit / max_loss) × (365 / DTE) × 100.

  - Reward/risk ratio = max_profit / max_loss.

- Rank candidates by annualized return descending.

- Return top 10 candidates with all details.

Method: enter_condor(self, symbol: str, account_equity: float, risk_pct: float = 2.0, width: int = 5) -> dict:

- Fetch current chain. Select expiration: nearest monthly with 30-45 DTE.

- Select short put: closest to 0.16 delta. If no exact match, use the strike with delta closest to -0.16.

- Select long put: short_put_strike - width. Verify this strike exists in the chain. If not, use closest available.

- Select short call: closest to 0.16 delta on call side.

- Select long call: short_call_strike + width.

- Calculate net credit from mid prices of all four legs.

- Size: max_contracts = floor(account_equity × risk_pct / max_loss_per_contract). Cap at configurable max (default 10).

- Collect all Greeks for the combined position:

  - Net delta = delta_short_put + delta_long_put + delta_short_call + delta_long_call (all with correct signs: short = negative of the option's delta from buyer's perspective for calls, positive for puts... actually: short put delta is positive, long put delta is negative, short call delta is negative, long call delta is positive. Net should be near zero for a balanced condor.)

  - Net gamma = sum of all gammas with signs (short positions have negative gamma contribution).

  - Net theta = sum of all thetas (short positions contribute positive theta — time decay works for you).

  - Net vega = sum of all vegas (short positions contribute negative vega — you want IV to fall).

- Record entry: timestamp, symbol, all four strikes, expiration, DTE, net_credit, max_profit, max_loss, breakevens, all Greeks, IV environment at entry, regime at entry (if available).

- Log to trade journal.

- Return full trade record.

Method: manage_condor(self, trade: dict, current_chain: dict) -> dict:

- Recalculate current value of all four legs using current mid prices.

- Current P&L = (entry_credit - current_cost_to_close) × 100 × contracts.

  - current_cost_to_close = long_put_mid + long_call_mid - short_put_mid - short_call_mid (what you'd pay to buy back shorts and sell longs).

  Wait, actually: to close an iron condor you buy back the shorts and sell the longs.

  - cost_to_close = short_put_ask + short_call_ask - long_put_bid - long_call_bid (conservative using natural prices).

  - Or use mids for a more accurate estimate: cost_to_close = short_put_mid + short_call_mid - long_put_mid - long_call_mid.

  - P&L = (entry_net_credit - cost_to_close) × 100 × contracts.

- current_pnl_pct = P&L / max_profit × 100 (as percentage of max profit).

Management checks (in priority order):

CHECK 1 — PROFIT TARGET: If current_pnl_pct >= 50% (configurable):

→ Action: CLOSE. Reasoning: "Profit target reached (X% of max profit). Close to lock in gains and free up capital for next trade."

CHECK 2 — TIME EXIT: If DTE <= 21 (configurable):

→ Action: CLOSE regardless of P&L. Reasoning: "DTE is X — gamma risk is too high to hold. Close and re-enter next cycle if setup still exists."

CHECK 3 — PUT SIDE TESTED: If underlying_price <= short_put_strike × 1.02 (within 2% of short put):

→ Evaluate rolling options:

  a. Can we roll the put spread down by 1-2 strikes AND out to next monthly for a net credit? Calculate the credit/debit of closing current put spread and opening new one.

  b. If net credit available → Action: ROLL_PUT. Provide new strikes and net credit.

  c. If no credit (would cost money to roll) → Action: CLOSE_PUT_SIDE. Close just the put spread, leave call spread open (becomes a bear call spread). Reasoning: "Put side tested, cannot roll for credit. Closing put spread to reduce risk."

  d. If both sides are near being tested → Action: CLOSE_ALL. Reasoning: "Both sides under pressure. Close entire condor."

CHECK 4 — CALL SIDE TESTED: If underlying_price >= short_call_strike × 0.98 (within 2% of short call):

→ Same logic as CHECK 3 but mirrored for the call side.

CHECK 5 — EARNINGS APPROACHING: If earnings date is within 7 days:

→ Action: CLOSE. Reasoning: "Earnings within X days. Binary event risk — close to avoid gap through strikes."

CHECK 6 — IV COLLAPSED: If current IVR is more than 20 points below IVR at entry:

→ Action: CLOSE. Reasoning: "IV has contracted significantly since entry (IVR dropped from X to Y). The vol contraction edge has been captured. Close."

CHECK 7 — MAX LOSS APPROACHING: If current_pnl_pct <= -150% (losing more than 1.5x the max profit):

→ Action: CLOSE. Reasoning: "Loss exceeds 1.5x max profit potential. Cut the loss."

DEFAULT: → Action: HOLD. Reasoning: "No management triggers hit. Condor is within normal range."

Return: action (CLOSE, HOLD, ROLL_PUT, ROLL_CALL, CLOSE_PUT_SIDE, CLOSE_CALL_SIDE, CLOSE_ALL), reasoning, current_pnl, current_greeks, new_strikes (if rolling).

Method: roll_side(self, trade: dict, side: str, current_chain: dict) -> dict:

- side = "put" or "call".

- Close existing spread on that side (buy back short, sell long).

- Open new spread: same DTE (next monthly if current is < 21 DTE), move short strike 1-2 strikes further OTM.

- Must be for a net credit. If the roll costs money, return None (don't roll).

- Calculate new max loss, new breakevens, new net credit.

- Update trade record with new strikes.

- Log roll to journal as an adjustment.

--- README.md ---

# Iron Condor Trading System

## What This Is

A complete, systematic iron condor trading system. Scans for high-IV candidates, structures optimal condors, manages positions with mechanical rules, tracks performance, and displays everything in an interactive dashboard. Optionally integrates with an HMM regime detector for regime-aware sizing.

## What an Iron Condor Is

An iron condor sells an OTM put spread and an OTM call spread simultaneously on the same underlying and expiration. You collect premium from both sides and profit when the stock stays between your short strikes until expiration. Max profit = net credit collected. Max loss = wing width minus net credit. It's a market-neutral, defined-risk income strategy.

## Quick Start

1. pip install -r requirements.txt

2. python main.py scan (find candidates)

3. python main.py enter SPY (paper-enter a condor)

4. python main.py manage (check management recommendations)

5. python main.py dashboard (launch full dashboard)

## Strategy Rules

- Entry: IVR > 50%, no earnings within 14 days, monthly expiration, 30-45 DTE.

- Strikes: short legs at ~0.16 delta (1 standard deviation OTM), wings at $5 width.

- Sizing: max 2% of equity risk per condor.

- Management: close at 50% profit, close at 21 DTE, roll tested side for credit, close if earnings approaching.

- Regime: reduce size in bear regimes, skip entry in crash regimes.

## Configuration

All parameters in config.yaml. See inline comments for each setting.

## Limitations

- Paper trading only — no broker integration. You place orders manually.

- yfinance data is delayed ~15 minutes. Greeks are estimates.

- IV rank uses HV approximation (see Polygon version for accurate IVR).

- This is not financial advice. Past performance does not predict future results.

Method: manage_spread(self, trade: dict, current_chain: dict) -> dict:

- Calculate current P&L from current mid prices.

- current_pnl_pct = realized P&L as percentage of max profit.

CHECK 1 — PROFIT TARGET: P&L >= 50% of max profit → CLOSE.

CHECK 2 — TIME EXIT: DTE <= 21 → CLOSE.

CHECK 3 — TESTED (bull put): price <= short_put_strike × 1.02 →

  - Try roll down and out for credit. If credit → ROLL. If no credit → CLOSE.

CHECK 3 — TESTED (bear call): price >= short_call_strike × 0.98 →

  - Try roll up and out for credit. If credit → ROLL. If no credit → CLOSE.

CHECK 4 — EARNINGS: within 7 days → CLOSE.

CHECK 5 — IV COLLAPSED: IVR dropped > 20 points from entry → CLOSE.

CHECK 6 — MAX LOSS: current loss > width × 100 × 0.8 (approaching max loss) → CLOSE.

CHECK 7 — DIRECTIONAL BIAS CHANGED: re-run directional analysis. If bias flipped (was bullish, now bearish for a bull put spread) → CLOSE.

DEFAULT → HOLD.

Return: action, reasoning, pnl, greeks.

--- strategy/strike_optimizer.py ---

Class: StrikeOptimizer

Method: optimal_strike(self, symbol, shares, cost_basis, iv_environment, regime=None) -> dict:

- This is the core optimization engine. It selects the best covered call to sell based on multiple factors.

SELECTION LOGIC:

1. ABSOLUTE RULE: never sell a call with strike BELOW cost_basis. This locks in a loss if called away. The strike must be above cost basis. If no strikes above cost basis have decent premium (< $0.20), skip this month — don't sell.

2. IV ENVIRONMENT ADJUSTMENT:

   - Very high IV (IVR > 80%): use 0.20 delta (further OTM). Premium is rich even far out. Preserve maximum upside.

   - High IV (50-80%): use 0.25 delta. Standard premium selling.

   - Normal IV (30-50%): use 0.30 delta. Closer strike needed for decent premium.

   - Low IV (< 30%): use 0.35 delta or skip. Premium is too thin to bother. Alternatively, sell a 45-60 DTE call instead of 30 DTE to capture more premium.

3. REGIME ADJUSTMENT (if available):

   - Bull regime: use lower delta (0.15-0.20). Don't cap upside in a strong trend. Give the stock room to run.

   - Neutral regime: standard (0.25-0.30). Range-bound = collect premium.

   - Bear regime: DO NOT sell covered calls in a confirmed bear. The stock is falling — you don't want to cap any potential recovery. Exception: if the stock has already fallen significantly and you want to reduce cost basis, sell far OTM (0.10 delta) calls.

   - Crash: absolutely no covered calls.

4. TECHNICAL ADJUSTMENT:

   - If RSI > 70 (overbought): move strike further OTM (reduce delta by 0.05). Stock may pull back — don't want to get called away at the top.

   - If RSI < 30 (oversold): move strike closer (increase delta by 0.05). Stock is beat up — collect more premium while it bases.

5. EXPIRATION SELECTION:

   - Default: nearest monthly with 30-45 DTE.

   - In low IV: extend to 45-60 DTE for more premium.

   - Never sell weeklies for covered calls (gamma risk, less premium per day).

6. CALCULATE AND RETURN:

   - Selected strike, expiration, DTE, premium (mid price), delta.

   - annualized_return = (premium / cost_basis) × (365 / DTE) × 100.

   - if_called_return = ((strike - cost_basis + premium) / cost_basis) × 100. Total return if shares are called away.

   - downside_protection = premium / current_price × 100. How much cushion the premium provides.

   - adjusted_cost_basis = cost_basis - premium. New effective cost basis after collecting premium.

   - reasoning: text explaining why this strike was selected (IV level, regime, technicals).

Class: EntrySignalEngine

Method: evaluate(self, symbol: str) -> dict:

- Pull technicals for the symbol.

- Generate a composite entry signal for both bullish and bearish setups.

BULLISH SIGNAL CRITERIA (all must be true for "strong_buy", 3+ for "buy", 2 for "weak_buy"):

BEARISH SIGNAL CRITERIA (mirror):

5. Volume > 20-day average.

  - Bull regime + bearish signal → REJECT. Don't fight the trend.

  - Bear regime + bearish signal → CONFIRM.

  - Bear regime + bullish signal → REJECT.

  - Neutral regime → allow either direction but reduce conviction to "moderate" max.

--- strategy/debit_spread.py ---

Class: DebitSpreadStrategy

Method: scan_candidates(self, universe: list) -> list:

- Universe: [SPY, QQQ, IWM, AAPL, MSFT, GOOGL, AMZN, NVDA, META, TSLA, AMD, JPM, V, MA, COST, UNH].

  4. No earnings within 7 days (unless intentionally playing earnings — separate flag).

  - Bullish → bull call spread: long call at ATM or slightly ITM (0.50-0.55 delta), short call at OTM (strike = long + width).

  - Bearish → bear put spread: long put at ATM or slightly ITM (0.50-0.55 delta), short put at OTM (strike = long - width).

- Calculate: net debit, max profit (width - debit) × 100, max loss (debit × 100), breakeven, reward/risk ratio (max_profit / max_loss), probability_of_profit (delta of long leg).

- Rank by: signal_strength first, then reward/risk ratio.

- Return top 10 candidates.

Method: enter_spread(self, symbol, direction, account_equity, risk_pct=2.0, width=3) -> dict:

- direction: "bullish" → bull call spread. "bearish" → bear put spread.

- Fetch chain. Select nearest monthly with 30-60 DTE (LONGER than income strategies to fight theta).

- For bullish: long call at closest to 0.50-0.55 delta. Short call = long_strike + width.

- For bearish: long put at closest to 0.50-0.55 delta. Short put = long_strike - width.

- Calculate net debit from mid prices.

- Size: max_contracts = floor(equity × risk_pct / (debit × 100)). Cap at 10.

- Position Greeks: net delta (positive for bull call, negative for bear put), net gamma, net theta (negative — time works against you), net vega.

- Record with: entry_signal details, technicals at entry, IV environment, regime.

- Return trade record.

Method: manage_spread(self, trade, current_chain) -> dict:

- Current value = current mid of long leg - current mid of short leg.

- P&L = (current_value - entry_debit) × 100 × contracts.

- pnl_pct_of_max = P&L / max_profit × 100.

- Also track P&L as % of debit: pnl_debit_pct = (current_value - entry_debit) / entry_debit × 100.

CHECK 1 — PROFIT TARGET: pnl_pct_of_max >= 50% → CLOSE. "50% of max profit captured. Book it."

CHECK 2 — LOSS LIMIT: pnl_debit_pct <= -50% → CLOSE. "Lost 50% of the debit. Cut the loss." This is a hard stop — no exceptions.

CHECK 3 — TIME EXIT: DTE <= 14 → CLOSE regardless. "Theta acceleration is destroying the spread. Close and re-evaluate." (14 DTE, not 21, because debit spreads are hurt more by theta.)

CHECK 5 — REGIME CHANGE: if regime was bull at entry and has shifted to bear → CLOSE bullish spreads. Vice versa.

CHECK 6 — EARNINGS: within 3 days → CLOSE unless this was an intentional earnings play.

CHECK 7 — MOMENTUM DIED: if ADX has dropped below 20 (was above 25 at entry) → CLOSE. "Trend has died. The directional bet no longer has a setup."

--- engine/black_scholes.py ---

Full Black-Scholes implementation. Inputs: S, K, T, r, sigma, option_type, q.

- d1 = (ln(S/K) + (r-q+sigma²/2)*T) / (sigma*sqrt(T)), d2 = d1 - sigma*sqrt(T)

- call_price, put_price, delta, gamma, theta (daily), vega (per 1%), rho (per 1%), all_greeks.

- Edge cases: T<=0 return intrinsic, T<0.001 use 0.001, sigma<=0 raise error.

--- engine/iv_solver.py ---

--- strategy/setups.py ---

Class: SetupDetector

This is the core engine that detects three types of momentum setups. Each returns a setup dict or None if no setup exists.

Method: detect_opening_range_breakout(self, symbol) -> dict or None:

- Pull intraday bars. Identify the opening range (first 30 minutes: 9:30-10:00 AM ET).

- opening_range_high = max(high) during 9:30-10:00.

- opening_range_low = min(low) during 9:30-10:00.

- opening_range_size = high - low.

- BULLISH BREAKOUT: current price > opening_range_high AND volume_ratio > 1.5 (volume confirms).

- BEARISH BREAKOUT: current price < opening_range_low AND volume_ratio > 1.5.

- Filter: opening_range_size must be reasonable (0.3% - 2.0% of price). Too small = noise. Too large = already moved.

- Filter: ADX > 20 (some trend exists at the daily level).

- If setup detected, return: setup_type ("orb_bullish" or "orb_bearish"), symbol, entry_price (current), stop_level (opposite side of opening range), target_level (opening_range_size projected from breakout point), risk_reward_ratio, volume_confirmation (bool), timestamp.

- If no setup, return None.

Method: detect_trend_line_break(self, symbol) -> dict or None:

- Pull daily bars (60 days).

- Identify key support/resistance: 50 SMA, 200 SMA, recent swing highs/lows (highest high and lowest low in last 20 days).

- BULLISH BREAK: price crosses above 50 SMA from below with volume_ratio > 1.3 AND MACD histogram just turned positive (crossover).

- BEARISH BREAK: price crosses below 50 SMA from above with volume_ratio > 1.3 AND MACD histogram just turned negative.

- Filter: this must be a FRESH break (happened within last 2 bars). Stale breaks don't count.

- Return: setup_type ("trend_break_bullish" or "trend_break_bearish"), symbol, entry_price, stop_level (other side of the SMA ± 0.5 ATR), target_level (1.5x the distance from entry to stop), risk_reward_ratio, sma_level, volume_confirmation.

Method: detect_power_hour(self, symbol) -> dict or None:

- Power hour = 3:00-4:00 PM ET. Only detect during this window.

- Pull intraday bars.

- BULLISH: price is above VWAP AND making new intraday highs in power hour AND volume is accelerating (last 30 min volume > previous 30 min).

- BEARISH: price is below VWAP AND making new intraday lows in power hour AND volume accelerating.

- This is the most aggressive setup — only valid for 0DTE or next-day expiry.

- Return: setup_type ("power_hour_bullish" or "power_hour_bearish"), symbol, entry_price, stop_level (VWAP or intraday pivot), target_level (continuation based on ATR), volume_acceleration (ratio).

Method: scan_all_setups(self, symbols: list) -> list:

- For each symbol, run all three detectors.

- Return list of all detected setups, sorted by risk_reward_ratio descending.

--- strategy/risk_controls.py ---

Class: MomentumRiskControls

Method: check_pre_trade(self, setup, portfolio, account_equity) -> dict:

- CHECK 1: is it within trading hours? (9:45 AM - 3:55 PM ET). No trades in first 15 minutes (spreads are wide, moves are erratic) or last 5 minutes (no time to manage).

- CHECK 2: max 3 momentum positions open.

- CHECK 3: total momentum exposure < 5% of equity.

- CHECK 4: no existing position in this symbol.

- CHECK 5: portfolio daily P&L check. If down > 2% today, no new trades. Cool off.

- CHECK 6: if using 0DTE options, verify the option has at least $0.10 bid (any less = no liquidity to exit).

- Return: approved (bool), reasons (list).

Method: check_circuit_breakers(self, portfolio) -> dict:

- BREAKER 1: daily momentum P&L < -3% of equity → HALT all momentum. Close everything.

- BREAKER 2: 3 consecutive losing momentum trades → HALT for rest of day.

- BREAKER 3: any single position lost 100% of premium → log but don't halt (expected for OTM options).

- Return: halt (bool), close_all (bool), reasoning.

Class: MomentumStrategy

Method: scan_setups(self, universe: list = None) -> list:

- For each setup, fetch chain data and identify the optimal option:

  - For ORB and trend break: 3-7 DTE options or debit spreads. ATM or slightly OTM (0.40-0.50 delta).

  - For power hour: 0DTE or 1DTE. ATM (0.45-0.55 delta). Must have tight bid-ask.

  - Alternative: debit spread with $2-3 width. Reduces cost but caps upside.

- Calculate for each: premium, delta, gamma, theta, breakeven, risk/reward.

- Return setups with option recommendations.

Method: enter_momentum_trade(self, symbol, setup_type, trade_type, account_equity, risk_pct=1.0) -> dict:

- trade_type: "long_call", "long_put", "bull_call_spread", "bear_put_spread".

- Run risk_controls.check_pre_trade(). If rejected, return rejection reason.

- Fetch chain. Select contract:

  - Long options: closest to 0.45-0.50 delta (near ATM for maximum gamma).

  - Debit spreads: long at 0.50 delta, short at long_strike ± width.

  - DTE: match to setup type. ORB/power_hour → 0-2 DTE. Trend break → 5-7 DTE.

- Size: MomentumPositionSizer. Max 1% risk per trade.

- Record: setup_type, entry_price, stop_level (from setup), target_level (from setup), all Greeks, IV at entry, regime, time_of_entry.

- Log to journal.

- Return trade record.

Method: manage_momentum_trade(self, trade, current_chain) -> dict:

- Calculate current P&L from current option mid price.

- pnl_pct = (current_mid - entry_premium) / entry_premium × 100.

FIRST: check circuit breakers. If triggered, close everything.

CHECK 1 — PROFIT TARGET: pnl_pct >= 50% → CLOSE.

  For strong momentum (volume still accelerating, price making new extremes): use 75% target. Let it run.

CHECK 2 — STOP LOSS: pnl_pct <= -50% → CLOSE. "Cut the loss. Premium has lost half its value."

  This is a HARD stop. No exceptions. Momentum trades that don't work quickly usually don't work at all.

CHECK 3 — UNDERLYING STOP: underlying price has hit the setup's stop_level → CLOSE. "Price hit the technical stop level. Setup has failed."

CHECK 4 — TIME STOP (0DTE): if DTE = 0 and time > 3:30 PM ET → CLOSE. "Closing 0DTE before final 30 minutes to avoid pin risk and widening spreads."

CHECK 5 — TIME STOP (weekly): if original DTE was 1-7 and current DTE <= 1 → CLOSE unless in profit. "Theta is destroying the position."

CHECK 6 — MOMENTUM DIED: if volume_ratio has dropped below 1.0 (below average) since entry → CLOSE. "Volume confirmation has evaporated. The move is losing steam."

CHECK 7 — REGIME CHANGE: if regime shifted from trending to choppy/neutral since entry → CLOSE. "Market regime no longer supports momentum plays."

DEFAULT → HOLD.

Return: action (CLOSE/HOLD), reasoning, current_pnl, current_greeks, time_in_trade.

--- strategy/leaps.py ---

Class: LeapsStrategy

Method: select_leaps(self, symbol, target_delta=0.80, min_dte=270) -> dict:

- Fetch LEAPS expirations. Pick the one closest to 12 months out (sweet spot: far enough for minimal theta, close enough that premium isn't astronomical).

- Find strike with delta closest to target_delta (0.80+). This is a deep ITM call.

- Calculate: intrinsic_value = max(0, stock_price - strike). extrinsic_value = mid_price - intrinsic_value. This extrinsic value is what decays over time — it's the "cost" of the LEAPS over buying shares.

- Also calculate: breakeven = strike + mid_price. Stock must be above this at expiry for the LEAP to profit.

- Return: symbol, strike, expiration, dte, mid_price, intrinsic, extrinsic, delta, theta_daily, breakeven, capital_required (mid × 100), shares_equivalent (delta × 100), capital_efficiency.

Method: enter_leaps(self, symbol, account_equity, risk_pct=8.0) -> dict:

- Run select_leaps.

- Size with LeapsPositionSizer.

- Record all details including fundamentals snapshot and IV environment.

- Log to journal.

Method: manage_leaps(self, trade, current_chain) -> dict:

- Track current value of LEAP from current chain mid price.

- P&L = (current_value - entry_cost) × 100 × contracts.

CHECK 1 — ROLL TRIGGER: DTE <= 180 (6 months remaining). Theta starts accelerating meaningfully.

  → Action: ROLL. Buy back current LEAP, buy new one at same delta target, 12 months out.

  - Calculate roll cost: cost of new LEAP - value of old LEAP.

  - If roll cost is reasonable (< 20% of position value): ROLL.

  - If roll cost is excessive (> 20%): CLOSE and re-evaluate. Maybe IV spiked.

CHECK 2 — DELTA DRIFT: if delta has fallen below 0.65 (stock dropped significantly):

  → Action: EVALUATE. Options:

  a. HOLD: if fundamentals still strong and regime is bullish. Delta will recover if stock recovers.

  b. CLOSE: if fundamentals deteriorated or bear regime. Take the loss.

  c. ROLL DOWN: buy back LEAP, buy new one at lower strike (closer to current price). Costs money but restores delta.

CHECK 3 — BIG WINNER: unrealized P&L > 50% of cost basis:

  → Action: HOLD but SET TRAILING STOP. If P&L drops back to 30%, close. Let winners run but protect gains.

CHECK 4 — STOCK GAP DOWN: stock has dropped > 15% from LEAPS entry price:

  → Action: CLOSE. "Significant adverse move. Position has lost substantial value. Cut losses."

CHECK 5 — IV SPIKE: if current LEAP IV is > 1.5× IV at entry:

  → Action: EVALUATE closing. LEAP value may have increased from IV alone — consider taking profit on the IV expansion even if stock hasn't moved much.

DEFAULT → HOLD.

--- strategy/pmcc.py ---

Class: PMCCStrategy (Poor Man's Covered Call)

Method: select_short_call(self, symbol, leaps_trade, iv_environment, regime=None) -> dict:

- This selects the short call to sell against the LEAP.

- ABSOLUTE RULE: short call strike must be ABOVE the LEAP's breakeven price (strike + entry_cost). If called away at the short strike, the LEAP must be profitable. Otherwise you lock in a loss.

- Delta target by IV environment:

  - High IV (IVR > 50%): sell 0.20 delta (further OTM, richer premium).

  - Normal IV: sell 0.25 delta.

  - Low IV (IVR < 30%): sell 0.30 delta or skip (premium too thin).

- Regime adjustment:

  - Bull: 0.15-0.20 delta (preserve upside).

  - Neutral: 0.25-0.30 delta (standard).

  - Bear: DON'T sell short calls. The stock is falling — you don't want to add a short delta position on top of a long LEAP that's losing value.

- DTE: 30-45 days, monthly.

- Calculate: premium, annualized_return (premium / leaps_cost × 365/DTE), max_profit_if_called (short_strike - leaps_strike - leaps_entry_cost + short_premium).

- Return all details.

Method: enter_pmcc_call(self, leaps_trade, account_equity) -> dict:

- Calls per LEAP: 1 short call per LEAP contract. Same number of contracts.

- Record short call details linked to the LEAPS trade_id.

- Log to journal.

Method: manage_pmcc_call(self, short_call_trade, leaps_trade, current_chain) -> dict:

- Current P&L of the short call.

CHECK 1 — PROFIT TARGET: call P&L >= 50% → CLOSE call. Sell new one for next cycle.

CHECK 2 — TIME EXIT: DTE <= 7 → CLOSE call, sell new one.

CHECK 3 — STOCK ABOVE SHORT STRIKE:

  - Option A: LET ASSIGN. Calculate total return if called away. If good → LET_ASSIGN.

    But wait — for PMCC, assignment means you need to exercise the LEAP (or close both). This costs the extrinsic value remaining in the LEAP. Only let assign if extrinsic value is small (< 5% of LEAP value).

  - Option B: ROLL UP AND OUT. Buy back call, sell higher strike at next monthly. → ROLL.

  - Recommend ROLL in most cases for PMCC since exercising the LEAP wastes remaining time value.

CHECK 4 — STOCK DROPPED: call is worth < $0.10 → CLOSE. Don't sell another until stock stabilizes.

CHECK 5 — EARNINGS within 7 days → CLOSE call.

DEFAULT → HOLD.

Class: ExpectedMoveCalculator

Method: expected_move_from_straddle(symbol, expiration) -> dict:

- The ATM straddle price represents the market's expected move in either direction.

- Pull ATM call and put for the given expiration.

Method: implied_vs_historical(symbol, expiration) -> dict:

- Compare expected_move_from_straddle vs historical_moves.

- If implied > avg_historical × 1.2: options are OVERPRICED for the expected move → favor SELLING vol.

- If implied < avg_historical × 0.8: options are UNDERPRICED → favor BUYING vol (our strategy).

- Return: implied_move, historical_avg, ratio, recommendation ("buy_vol", "sell_vol", "neutral").

Class: LongVolStrategy

Method: enter_straddle(self, symbol, account_equity, risk_pct=3.0) -> dict:

- Select ATM strike (closest to current price). DTE: 30-60 days, monthly.

- Buy 1 ATM call + 1 ATM put at the same strike.

- Total debit = call_mid + put_mid.

- Size: contracts = floor(equity × risk_pct / (total_debit × 100)).

- Position Greeks: net delta ≈ 0 (market neutral), net gamma POSITIVE (you benefit from moves), net theta NEGATIVE (time works against you), net vega POSITIVE (you benefit from IV expansion).

- Record: all details + catalyst_info + implied_vs_historical_ratio + iv_environment.

Method: enter_strangle(self, symbol, account_equity, risk_pct=2.0, delta_target=0.30) -> dict:

- Buy OTM put at -delta_target + buy OTM call at +delta_target. Different strikes.

- Cheaper than straddle but needs a bigger move.

- DTE: 45-90 days (MORE time since OTM options need bigger moves).

- Same sizing, Greeks, and logging as straddle.

Method: manage_long_vol(self, trade, current_chain) -> dict:

- Calculate current value of both legs from current mids.

- P&L = (current_call_mid + current_put_mid - entry_debit) × 100 × contracts.

- pnl_pct = P&L / (entry_debit × 100 × contracts) × 100.

- Also track: which leg is winning (call or put) and by how much.

CHECK 1 — WINNING LEG PROFIT: if the winning leg has gained > 100% of its entry value:

  → Action: CLOSE_WINNING_LEG. "The winning leg has doubled. Lock in profit on this side."

  → Keep the losing leg as a "lottery ticket" — it costs nothing extra (already paid for) and can still pay off if the stock reverses.

CHECK 2 — TOTAL POSITION PROFIT: pnl_pct >= 50% → CLOSE_BOTH. "Combined position is profitable by 50%. Take the win."

CHECK 3 — TIME EXIT: DTE <= 21 → CLOSE_BOTH. "Theta is accelerating. The position is decaying too fast to hold."

CHECK 4 — POST-CATALYST: if the catalyst has passed (earnings announced, FOMC done) AND the position is within 5% of entry value → CLOSE_BOTH. "Catalyst passed without sufficient move. IV will crush. Exit before vega collapse."

CHECK 5 — IV CRUSH: if current IV is more than 30% lower than IV at entry → CLOSE_BOTH. "Significant IV contraction since entry. The vega loss is overwhelming the position."

CHECK 6 — TOTAL LOSS: pnl_pct <= -50% → CLOSE_BOTH. "Cut the loss at 50% of debit."

CHECK 7 — PRE-CATALYST PROFIT: if a catalyst is 1-3 days away AND pnl_pct >= 20% (from IV expansion alone before the event):

  → Action: EVALUATE. Options:

  a. CLOSE_BOTH: lock in the IV expansion profit before the event. Safe choice.

  b. HOLD: ride through the catalyst for a bigger move. Higher risk/reward.

  → Present both options with P&L scenarios.

--- strategy/pre_earnings.py ---

Class: PreEarningsStrategy

Method: enter_pre_earnings(symbol, account_equity, risk_pct=2.0) -> dict:

- PURPOSE: capture IV EXPANSION in the days before earnings. IV typically rises 5-14 days before earnings. Enter early, exit before the event.

- Structure: buy straddle or strangle 7-14 days before earnings.

- DTE: use the expiration AFTER earnings (so you have time value through the event, but plan to exit before).

- Position will profit from IV expansion even if stock doesn't move.

- Size: standard. Risk_pct 2%.

- CRITICAL EXIT RULE: close BEFORE the earnings announcement. This is an IV expansion play, not an earnings direction bet.

- Record entry with iv_at_entry (should be relatively low, will expand into earnings).

Method: manage_pre_earnings(trade, current_chain) -> dict:

- Track IV change since entry.

CHECK 1: IV has expanded > 20% from entry → CLOSE. "IV expansion captured. Exit before event."

CHECK 2: earnings is tomorrow (1 day away) → CLOSE regardless. "Must exit before binary event."

CHECK 3: P&L >= 15-25% → CLOSE. "Modest profit from IV expansion. Lock it in."

CHECK 4: 3 days before earnings and P&L is negative → CLOSE. "IV hasn't expanded as expected. Cut loss before earnings risk."

DEFAULT: HOLD (but never through the event).

--- strategy/through_earnings.py ---

Class: ThroughEarningsStrategy

Method: enter_earnings_condor(symbol, account_equity, risk_pct=2.0) -> dict:

- PURPOSE: SELL premium to capture IV crush after earnings. The stock moves, but not as much as options are pricing.

- Only enter if implied_vs_realized ratio > 1.2 (options are overpriced).

- Structure: iron condor with short strikes at the implied move distance.

  - Short put at (current_price - implied_move).

  - Short call at (current_price + implied_move).

  - Wings: $2-3 wide.

- This positions you OUTSIDE the expected move on both sides.

- Enter: 1-3 days before earnings (to capture max pre-event IV).

- Size: SMALLER than normal condors. Risk_pct 1.5% max. Earnings are binary — the condor can go from max profit to max loss instantly.

Method: enter_earnings_straddle(symbol, account_equity, risk_pct=2.0) -> dict:

- PURPOSE: BUY premium betting on a bigger move than expected.

- Only enter if implied_vs_realized ratio < 0.8 (options are underpriced).

- Structure: ATM straddle.

- Enter: 1-3 days before earnings.

- Size: risk_pct 2%. Max loss = total debit.

Method: manage_through_earnings(trade, current_chain) -> dict:

- PRE-EARNINGS (trade is still open, earnings hasn't happened yet):

  - HOLD. Wait for the event.

- POST-EARNINGS (earnings has been announced — detected by gap or time):

  CHECK 1: morning after earnings → CLOSE. All earnings plays close by 10:00 AM the day after. No holding through additional sessions. The event is done, IV has crushed, the trade is either a winner or loser.

  - Exception: if condor and stock is BETWEEN short strikes (winning) → can hold for 1 more day to capture remaining time value. But close by end of that day.

- Never hold an earnings play past 10 AM the morning after the announcement.

- Daily circuit breaker: if earnings plays lost > 4% of equity today, halt.
