# AI For Quant Trading Context

The Jarvis Quant project uses AI as a system builder, analyst, monitor, reviewer, and supervisor.

AI is not treated as magic alpha.

## Correct use of AI

AI can help with:
- coding
- testing
- backtesting infrastructure
- dashboards
- data checks
- research memos
- failure diagnosis
- strategy documentation
- safety reviews
- workflow automation

AI can also act as a non-deterministic analyst:
- summarize market context
- compare setups
- explain failures
- generate hypotheses
- review option-chain quality
- review strategy assumptions

## Incorrect use of AI

AI should not:
- hallucinate performance
- invent backtest results
- place live trades
- bypass risk controls
- override deterministic gates
- treat social-media trading claims as proof
- assume a strategy works without data

## Core lesson

Profit does not come from “AI trading” by itself.

The edge must come from:
- data
- market structure
- risk control
- asymmetric payoff
- robust testing
- execution discipline
- human-approved promotion gates

## AI Pathways architecture

The system separates:

1. Deterministic systems:
   - rules
   - tests
   - backtests
   - scanners
   - paper simulation
   - reports

2. Non-deterministic AI analysts:
   - thesis review
   - market interpretation
   - code review
   - failure diagnosis
   - research assistance

The supervisor must keep those roles separate.
